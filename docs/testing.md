# Testing

## Running Tests

```bash
# Run all tests
npm test

# Run a specific test file
npx hardhat test test/DAO.test.ts

# Run tests against Sepolia (requires deployed contracts)
npm run test:sepolia

# Run with gas reporting
REPORT_GAS=true npm test
```

## FHE Mock vs Sepolia

Tests use `@fhevm/hardhat-plugin` which provides a mock FHE environment on the local Hardhat network. This means:

- **Local Hardhat** (`npm test`): FHE operations are mocked. All encrypted types work, but there is no real encryption — values are stored as mock handles. The `fhevm.isMock` flag is `true`.

- **Sepolia** (`npm run test:sepolia`): FHE operations use the real Zama coprocessor. You need `@zama-fhe/relayer-sdk` for encrypted input creation and user decryption. The `fhevm.isMock` flag is `false`.

### Gating Tests by Environment

Tests that require FHE operations check `fhevm.isMock`:

```typescript
import { fhevm } from "hardhat";

describe("FHE-dependent tests", function () {
  beforeEach(function () {
    if (!fhevm.isMock) this.skip();
  });

  it("should work with encrypted values", async function () {
    // This test only runs on local Hardhat with mock FHE
  });
});
```

For tests that should only run on Sepolia (real FHE), invert the check:

```typescript
beforeEach(function () {
  if (fhevm.isMock) this.skip();
});
```

## Test Categories

### Pure Solidity Tests (run everywhere)

These tests don't touch FHE operations and work on any network:

- Constructor validation (invalid args → revert)
- Permission checks (unauthorized → revert)
- View function error paths (invalid proposal ID → revert)
- ETH transfer execution
- Event emission

### FHE Mock Tests (local Hardhat only)

These tests use FHE operations via the mock:

- Token minting (creates encrypted balances)
- Delegation (modifies encrypted voting power)
- Multisig deployment (creates encrypted signer flags)

### Full Integration Tests (Sepolia only)

Full proposal lifecycle tests require:
- `fhevm.createEncryptedInput()` for encrypted calldata chunks
- `fhevm.userDecryptEuint()` for decrypting results
- KMS signatures for `finalize()` and `revealProposal()`

Example pattern for creating encrypted inputs:

```typescript
import { fhevm } from "hardhat";

const contractAddress = await voting.getAddress();
const encryptedInput = await fhevm
  .createEncryptedInput(contractAddress, signer.address)
  .add256(chunkValue)
  .encrypt();

await voting.createProposal(
  encryptedInput.handles,
  encryptedInput.inputProof,
  cancelKeyHash,
  { startDate: 0, endDate: 0 },
);
```

## Writing New Tests

1. Import from `"hardhat"` (not `"ethers"` directly):
   ```typescript
   import { ethers, fhevm } from "hardhat";
   ```

2. Import types from `"../types"` (generated by typechain):
   ```typescript
   import type { DAO, EncryptedGovernanceToken } from "../types";
   ```

3. Use `fhevm.isMock` to gate FHE-dependent tests.

4. For encrypted inputs, use `fhevm.createEncryptedInput()`.

5. For user decryption, use `fhevm.userDecryptEuint()` / `fhevm.userDecryptEbool()`.
